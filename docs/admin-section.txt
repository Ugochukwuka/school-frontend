<h2>API Endpoints</h2>

<h3>Student Management Endpoints</h3>

<ul>
    <li><strong>POST /api/admin/students</strong> - Add new student</li>
    <li><strong>GET /api/admin/viewstudents</strong> - View all students (with optional filters)</li>
    <li><strong>POST /api/admin/students/promote</strong> - Promote students to new class</li>
</ul>

<h3>Teacher Management Endpoints</h3>

<ul>
    <li><strong>POST /api/admin/teachers</strong> - Add new teacher</li>
    <li><strong>GET /api/admin/teachers</strong> - View all teachers</li>
    <li><strong>PUT /api/admin/teachers/{uuid}</strong> - Update teacher information</li>
    <li><strong>POST /api/admin/teachers/assign-subject</strong> - Assign subject to teacher</li>
    <li><strong>POST /api/admin/teachers/assign-class</strong> - Assign class to teacher</li>
    <li><strong>POST /api/admin/classes/{class_id}/subjects</strong> - Add subject to class</li>
</ul>

<h3>Fee Management Endpoints</h3>

<ul>
    <li><strong>POST /api/admin/fees</strong> - Add new fee</li>
    <li><strong>GET /api/admin/fees/unpaid</strong> - View unpaid fees</li>
    <li><strong>GET /api/admin/fees/part-paid</strong> - View partially paid fees</li>
    <li><strong>GET /api/admin/fees/paid</strong> - View completed payments</li>
    <li><strong>POST /api/admin/fees/pay</strong> - Process fee payment</li>
    <li><strong>GET /api/admin/fees/payment-history</strong> - View payment history</li>
    <li><strong>GET /api/admin/fees/obligations</strong> - View fee obligations</li>
</ul>

<h3>Results Management Endpoints</h3>

<ul>
    <li><strong>POST /api/results/enter</strong> - Enter student result</li>
    <li><strong>POST /api/results/view</strong> - View all results</li>
    <li><strong>PUT /api/results/update/{id}</strong> - Update result</li>
    <li><strong>GET /api/results/single/{id}</strong> - View single result</li>
</ul>

<h3>Bookshop Management Endpoints</h3>

<ul>
    <li><strong>POST /api/bookshop/books</strong> - Add new book</li>
    <li><strong>GET /api/bookshop/books</strong> - View all books</li>
    <li><strong>PUT /api/bookshop/books/{id}</strong> - Update book</li>
    <li><strong>POST /api/bookshop/books/assign</strong> - Assign book to class</li>
    <li><strong>GET /api/bookshop/books/requirements</strong> - View book requirements</li>
    <li><strong>GET /api/bookshop/books/status</strong> - View book status</li>
    <li><strong>GET /api/bookshop/sales</strong> - View all books sold</li>
</ul>

<h3>Announcements Endpoints</h3>

<ul>
    <li><strong>POST /api/admin/announcements</strong> - Add new announcement</li>
    <li><strong>GET /api/admin/announcements</strong> - View all announcements</li>
</ul>

<h2>Best Practices</h2>

<ul>
    <li>Regularly review and update student and teacher records to maintain data accuracy</li>
    <li>Process fee payments promptly and keep accurate financial records</li>
    <li>Use filtering and search features effectively to quickly locate specific records</li>
    <li>Verify book assignments match curriculum requirements before publishing</li>
    <li>Keep announcements clear, concise, and targeted to appropriate audiences</li>
    <li>Review dashboard statistics regularly to monitor system health and usage</li>
    <li>Maintain proper access controls and user permissions for security</li>
    <li>Back up important data regularly and maintain audit trails for critical operations</li>
</ul>
