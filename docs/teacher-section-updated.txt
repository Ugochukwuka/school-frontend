<h2>API Endpoints</h2>

<h3>Attendance Endpoints</h3>

<ul>
    <li><strong>POST /api/teacher/attendance/mark</strong> - Mark attendance for students</li>
    <li><strong>GET /api/teacher/attendance/summary</strong> - Get attendance summary</li>
    <li><strong>PUT /api/teacher/attendance/update</strong> - Update attendance records</li>
</ul>

<h3>Results Endpoints</h3>

<ul>
    <li><strong>POST /api/results/enter</strong> - Enter student results</li>
    <li><strong>POST /api/results/view</strong> - View all results (requires session_id and term_id)</li>
    <li><strong>GET /api/results/single/{id}</strong> - View single result</li>
    <li><strong>PUT /api/results/update/{id}</strong> - Update result</li>
</ul>

<h3>Class and Student Endpoints</h3>

<ul>
    <li><strong>GET /api/teacher/classes/students</strong> - Get students in assigned classes</li>
    <li><strong>GET /api/teacher/formclass/students</strong> - Get students in form class</li>
    <li><strong>GET /api/teacher/subjects</strong> - Get assigned subjects</li>
</ul>

<h2>Dark Mode</h2>

<p>The teacher dashboard features a built-in dark mode interface that provides a comfortable viewing experience with dark backgrounds and light text. Dark mode is automatically enabled for all teacher users.</p>

<h2>Best Practices</h2>

<ul>
    <li>Mark attendance daily to maintain accurate records</li>
    <li>Enter results promptly after assessments to keep records up-to-date</li>
    <li>Use filters effectively to find specific student information quickly</li>
    <li>Review attendance summaries regularly to identify patterns</li>
    <li>Update results when corrections are needed to maintain data accuracy</li>
</ul>
